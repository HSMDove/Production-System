"أنت الآن مهندس أنظمة خلفية (Backend Architect) وخبير في بناء منصات الـ SaaS. اكتشفنا ثغرة منطقية حرجة: واجهة الإعدادات حالياً 'وهمية' (Cosmetic)، حيث يتجاهل الخادم (Backend) اختيارات المستخدم ويقوم بتوجيه جميع الطلبات للنموذج الافتراضي.

بالإضافة إلى ذلك، نحتاج إلى تهيئة هيكلية قاعدة البيانات لتكون قابلة للإدارة والمراقبة الكاملة عبر لوحة تحكم خارجية (Retool).

المطلوب تنفيذه فوراً بناءً على النتائج الصارمة التالية فقط (أنت من يحدد التعديلات البرمجية وتحديثات الـ Schema اللازمة):

1. فرض المنطق الصارم لإعدادات الذكاء الاصطناعي (Strict API Enforcement):



النتيجة المطلوبة: يجب أن يكون الباك اند هو 'مصدر الحقيقة'. قبل تنفيذ أي طلب لـ 'فكري'، يجب على النظام قراءة إعدادات المستخدم.

إذا اختار المستخدم 'مفتاح API مخصص' وكان الحقل فارغاً أو المفتاح غير صالح، يجب أن يرفض النظام الطلب فوراً ويعيد رسالة خطأ واضحة: 'يرجى إدخال مفتاح API صحيح في الإعدادات'، ولا يقوم بالتحويل للنموذج الافتراضي أبداً.

إذا اختار 'النموذج الافتراضي'، يتم استخدام مفاتيح النظام.

2. توسيع خيارات أداة البحث (Web Search API Configuration):



النتيجة المطلوبة: تطبيق نفس خيارات الذكاء الاصطناعي على أداة 'البحث في الإنترنت'.

يجب أن تحتوي إعدادات البحث على خيارات: (النموذج الافتراضي للموقع، ومفتاح API مخصص).

يجب أن يعكس الباك اند هذا الاختيار بصرامة عند تشغيل أداة البحث، تماماً كما في النقطة الأولى.

3. تهيئة قاعدة البيانات للمراقبة الشاملة (Retool & Metrics Readiness):



النتيجة المطلوبة: تعديل الجداول (أو إنشاء جداول جديدة للمقاييس Metrics) لتسجيل نقاط البيانات التالية ليتم عرضها لاحقاً في Retool:

أ. تتبع الاستهلاك (Usage Tracking): تسجيل عدد الطلبات (Requests) والاستهلاك الفعلي لكل مستخدم، مع توضيح 'نوع المصدر' المستخدم في كل طلب (هل استخدم النموذج الافتراضي للموقع أم الـ API الخاص به؟).

ب. حالة النظام (System Health): حقل لتسجيل حالة 'فكري' (Online/Offline) وحالة أداة البحث لتسهيل مراقبتها.

ج. النشاط: تسجيل تاريخ 'آخر ظهور/نشاط' للمستخدم لمعرفة المستخدمين النشطين (Active Users).

4. نظام التحكم بالمميزات (Feature Flags & Global Controls):



النتيجة المطلوبة: تصميم هيكلية في قاعدة البيانات (مثلاً جدول system_settings) تسمح لي كمدير (من خلال Retool) بتفعيل أو تعطيل ميزات معينة في الموقع بالكامل فوراً (مثل: إيقاف 'فكري'، إيقاف تسجيل مستخدمين جدد، إيقاف أداة البحث). يجب أن يتحقق الباك اند من حالة هذه الميزات قبل تنفيذ أي طلب.

5. العزل الأمني الصارم (Tenant Isolation):



النتيجة المطلوبة: التأكد التام من أن كل هذه التعديلات لا تكسر العزل الأمني. لا يمكن لأي مستخدم استهلاك رصيد أو مفاتيح مستخدم آخر تحت أي ظرف.
